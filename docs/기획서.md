## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **제목** | 제주어 번역·감정분석 챗봇 |
| **한 줄 소개** | 제주어 문장을 입력하면 **표준어로 번역**하고 **감정을 분석**해 주며, 반대로 **표준어를 제주어로** 바꿔 주는 대화형 챗봇 |
| **분류** | NLP / 소멸위기언어 보존 / 대화형 서비스 |
| **대상 학기** | 1학기 번역기 제안 → 2학기 감정분류 → 통합 챗봇 |

### 추진 배경

제주어는 **유네스코가 지정한 소멸위기언어(2010년, "심각한 위기" 등급)**다. 표준어 중심 교육과 미디어 환경, 그리고 **세대 간 단절**로 인해 젊은 세대는 제주어를 듣거나 이해할 기회가 급격히 줄어들고 있다. 조부모 세대가 일상적으로 쓰는 제주어를 손주 세대가 알아듣지 못하는 일이 흔하며, 이는 언어를 넘어 **지역 정체성과 문화 기억의 단절**로 이어진다.

이런 상황에서 제주어를 **"이해 가능한 형태"로 즉시 변환**해 주고, 더 나아가 그 안에 담긴 **감정까지 읽어 주는** 도구는 학습·소통·콘텐츠·연구 모두에 실질적 가치를 가진다. 본 프로젝트는 1·2학기에 걸쳐 축적한 **제주어 번역기 제안**과 **제주어 감정분류(F1-Macro 0.84) 성과**를 하나의 사용 가능한 제품으로 통합하는 것을 목표로 한다.

## 2. 문제 정의 및 필요성

### 문제

1. **이해 장벽** — 제주어 원문은 어휘·어미·억양이 표준어와 크게 달라 비제주민·젊은 세대가 의미를 파악하기 어렵다.
2. **학습 수단 부재** — 제주어를 체계적으로 학습할 수 있는 디지털 도구가 부족하다. 사전은 있으나 *문장 단위 변환*을 즉시 제공하는 서비스가 드물다.
3. **감정·뉘앙스 손실** — 단순 번역만으로는 문장에 담긴 정서(슬픔, 분노, 기쁨 등)가 충분히 전달되지 않는다. 특히 방언은 정서 표현이 풍부해 감정 정보의 가치가 크다.
4. **성과의 파편화** — 번역과 감정분류가 각각 따로 연구되어, 사용자가 실제로 손에 쥐고 써 볼 수 있는 통합 제품이 없다.

### 필요성

| 관점 | 필요성 |
|------|--------|
| **문화 보존** | 소멸위기언어의 디지털 접근성을 높여 학습·재사용을 촉진 |
| **세대 소통** | 제주 출신 젊은 세대가 가족·지역 어른의 말을 이해하는 다리 역할 |
| **기술 가치** | 저자원(low-resource) 방언에 대한 번역·감정분석의 실증 사례 제공 |
| **활용성** | 관광·콘텐츠·교육·언어연구 등 다방면 응용 기반 마련 |

> sklearn TF-IDF 기반 베이스라인은 제주어 감정분류에서 F1-Macro **0.30**에 그쳤으나, 딥러닝 전환(KR-BERT 단일 0.76 → 앙상블 0.84)으로 실용 수준의 성능을 확보했다. 즉 본 프로젝트는 **이미 검증된 기술 토대** 위에서 통합 제품화를 추진한다.

## 3. 목표 및 범위

### 목표

제주어 ↔ 표준어 **양방향 번역**과 제주어 **감정분석(7종)**을 **하나의 채팅 화면**에서 end-to-end로 동작시키는 데모 제품을 완성한다.

### MVP 기능 (4종)

| # | 기능 | 입력 → 출력 |
|---|------|------------|
| 1 | **제주어 → 표준어 번역** | 제주어 문장 → 표준어 문장 |
| 2 | **표준어 → 제주어 번역** | 표준어 문장 → 제주어 문장 (양방향) |
| 3 | **감정 분석** | 제주어 문장 → 7종 감정 라벨 + 신뢰도 |
| 4 | **채팅 UI** | 위 1~3 기능을 한 화면에서 사용하는 대화형 인터페이스 |

**감정 7종:** 중립 · 슬픔 · 행복 · 분노 · 놀람 · 공포 · 혐오

### 비목표 (YAGNI — MVP에서 명시적으로 제외)

- **음성 입출력(STT/TTS)** — 텍스트 입력에 집중
- **로그인 / 계정 관리** — 익명 단발 사용으로 충분
- **멀티턴 문맥 기억** — 각 입력을 독립 처리
- **모바일 네이티브 앱** — 웹 기반 단일 페이지로 제공

> 위 항목은 모두 *향후 과제*로 남기며, 기능 확장이 용이하도록 모듈식으로 설계한다.

### 성공 지표

| 영역 | 지표 | 목표 |
|------|------|------|
| 감정 분류 | Test F1-Macro | **0.84** (발표본 재현, 진행 중) |
| 번역 | BLEU / chrF | 테스트셋 측정·보고, 베이스라인 대비 향상 입증 |
| 통합 데모 | end-to-end | 제주어 입력 → 번역 + 감정이 한 화면에서 동작 |

## 4. 대상 사용자 및 사용 시나리오

### 대상 사용자

| 사용자 | 핵심 니즈 |
|--------|-----------|
| **제주어 학습자** | 제주어 문장의 정확한 표준어 의미를 빠르게 확인 |
| **제주 출신 젊은 세대** | 가족·지역 어른의 제주어를 이해, 또는 자신이 제주어로 표현 |
| **콘텐츠/관광 종사자** | 제주어 자막·안내·홍보 문구 작성 및 검수 |
| **언어 연구자** | 제주어 표현과 정서 분포를 데이터로 관찰 |

### 사용 시나리오

#### 시나리오 1 — 손주가 할머니의 말을 이해하기 (제주어 → 표준어 + 감정)

> 제주 출신 대학생 A는 할머니가 보낸 메시지의 뜻을 정확히 알고 싶다.

```
[방향 토글: 제주어 → 표준어]

사용자(제주어):  속솜허영 밥이라도 챙겨 먹읍서.
─────────────────────────────────────────
챗봇 번역(표준어): 조용히 있으면서 밥이라도 챙겨 먹어요.
챗봇 감정 분석:   🙂 행복  (신뢰도 0.71)
                  걱정과 애정이 담긴 다정한 표현으로 분석됩니다.
```

A는 할머니의 말이 단순 지시가 아니라 **애정 어린 당부**임을 감정 뱃지로 확인한다.

#### 시나리오 2 — 관광 콘텐츠 제작자가 제주어 문구 만들기 (표준어 → 제주어)

> 관광 홍보 영상에 제주어 자막을 넣으려는 콘텐츠 PD B.

```
[방향 토글: 표준어 → 제주어]

사용자(표준어):  어서 오세요, 천천히 구경하세요.
─────────────────────────────────────────
챗봇 번역(제주어): 혼저 옵서, 놀멍 보멍 갑서.
```

B는 결과 문구를 자막 후보로 채택하고, 필요 시 다시 제주어 → 표준어로 역검수한다.

#### 시나리오 3 — 학습자가 정서 표현을 익히기 (제주어 → 표준어 + 감정)

> 제주어를 배우는 학습자 C가 문장의 의미와 정서를 함께 학습한다.

```
[방향 토글: 제주어 → 표준어]

사용자(제주어):  무사 경 햄시니? 속상허다게.
─────────────────────────────────────────
챗봇 번역(표준어): 왜 그렇게 하니? 속상하다.
챗봇 감정 분석:   😢 슬픔  (신뢰도 0.83)
```

C는 같은 표현을 여러 번 입력해 **표현–의미–감정**의 연결을 자연스럽게 학습한다.

## 5. 주요 기능 상세

### 5.1 제주어 → 표준어 번역
- 제주어 문장을 입력받아 의미가 통하는 표준어 문장으로 변환.
- 짧은 일상 발화 중심 데이터로 학습되어 구어체 변환에 강점.

### 5.2 표준어 → 제주어 번역 (양방향)
- 동일 모델이 방향 토글에 따라 역방향 변환 수행.
- 단일 모델에 **방향 prefix**를 부여해 두 방향을 함께 학습 → 표현 공유 및 데이터 효율 확보.

### 5.3 감정 분석 (7종 + 신뢰도)
- 입력 제주어 문장의 감정을 **중립/슬픔/행복/분노/놀람/공포/혐오** 7종으로 분류.
- 각 감정의 확률(신뢰도)을 함께 제공해, 단정이 아닌 **분포로 정서 제시**.
- 감정별 **이모지 + 색상 뱃지**와 **신뢰도 막대**로 직관적 표시.

### 5.4 채팅 UI
- 말풍선 기반 대화형 인터페이스. 사용자 입력과 챗봇 응답이 시간순으로 누적.
- **방향 토글**(제주어↔표준어)로 번역 방향 전환.
- 번역 결과 + 감정 뱃지가 한 응답에 함께 표시.
- 빌드 단계 없이 즉시 실행 가능한 가벼운 단일 페이지.

## 6. 기술 개요

> 상세 아키텍처·학습 절차·API 명세는 **개발설계서**로 위임한다. 여기서는 개괄만 다룬다.

### 전체 구성

```
[브라우저 채팅 UI]  ──HTTP(JSON)──>  [FastAPI 백엔드]
   정적 HTML/JS                         ├─ EmotionService  (감정 앙상블 추론)
   방향 토글, 감정 뱃지                   ├─ TranslationService (KoBART 추론)
                                        └─ /api/translate · /api/emotion · /api/chat
```

### 핵심 모델

| 서브시스템 | 모델 | 비고 |
|------------|------|------|
| **감정분석** | **Dual-Gated KR-BERT + KoELECTRA 앙상블** | 목표 Test F1-Macro **0.84**. 제주어/표준어를 shared KR-BERT로 인코딩 후 게이팅 융합, KoELECTRA와 softmax 평균 앙상블 |
| **번역** | **KoBART (`gogamza/kobart-base-v2`) 양방향** | 한국어 seq2seq를 병렬 코퍼스로 파인튜닝. 방향 prefix로 양방향 단일 모델 구성. BLEU/chrF로 평가 |
| **백엔드** | **FastAPI** | 시작 시 모델 1회 로드(MPS/CPU 자동), REST API + 정적 프론트 제공 |

### 데이터
- `merged_jeju_balanced_sorted.xlsx` — 제주어/표준어 병렬 코퍼스 약 **127k쌍**, 7클래스 거의 균형.
- 감정: Stratified 7:1:2 분할(seed=42 고정). 번역: 8:1:1 분할.

### 설계 원칙
- 3개 서브시스템(감정 · 번역 · 챗봇 앱)을 **독립 개발·테스트** 가능하도록 분리.
- 모델 아티팩트 경로는 설정으로 주입하며, 모델 미존재 시 **가능한 기능만 노출**(graceful degradation).
- 단일 GPU 환경을 고려해 GPU 작업(학습)은 **직렬 실행**.

## 7. 기대 효과 및 활용 방안

### 기대 효과

| 구분 | 효과 |
|------|------|
| **문화적** | 소멸위기 제주어의 디지털 접근성·학습성 제고, 세대 간 소통 지원 |
| **기술적** | 저자원 방언에 대한 양방향 번역 + 감정분석 통합 사례 제시 |
| **실용적** | 손에 쥐고 쓸 수 있는 end-to-end 데모로 연구 성과의 가시화 |
| **학술적** | TF-IDF(0.30) → 딥러닝 앙상블(0.84) 도약을 입증한 실증 결과 정리 |

### 활용 방안

- **교육** — 제주어 학습 보조 도구, 표현·정서 학습 콘텐츠.
- **관광/콘텐츠** — 제주어 자막·안내문·홍보 문구 작성 및 검수.
- **소통** — 가족·지역 어른과의 메시지 이해 도구.
- **연구** — 제주어 표현별 감정 분포 관찰, 방언 NLP 후속 연구의 베이스.
- **확장 기반** — 음성(STT/TTS), 멀티턴 대화, 모바일 앱 등 향후 기능의 토대.

## 8. 추진 일정

| 단계 | 기간 | 내용 | 산출물 |
|------|------|------|--------|
| **1단계** | 1학기 | 제주어 번역기 **제안** 및 데이터/접근 정의 | 번역기 제안서, 병렬 코퍼스 정리 |
| **2단계** | 2학기 | 제주어 **감정분류** 모델 개발 (KR-BERT → 앙상블) | F1-Macro 0.84 모델, 감정분류 보고서 |
| **3단계** | 통합기 (현재) | 번역 모델 학습 + 두 모델을 챗봇으로 **통합** | 통합 챗봇 데모, 기획서·개발설계서 |

### 통합기 세부 작업 순서 (단일 GPU 직렬화)

| 순서 | 작업 | GPU | 비고 |
|------|------|-----|------|
| 1 | 문서화(기획서·개발설계서) + 번역/앱 코드 스캐폴드 + 모델저장 코드 보강 | 비-GPU | 병렬 진행 |
| 2 | 감정 모델 학습 완료 → F1 0.84 검증 | GPU | 진행 중 |
| 3 | 번역 KoBART 학습 → BLEU/chrF 평가 | GPU | 감정 학습 종료 후 |
| 4 | 두 모델을 앱에 연결 → end-to-end 데모 | 비-GPU | |
| 5 | 리포트·문서 갱신, 최종 발표 준비 | 비-GPU | |

## 9. 위험 요소 및 대응

| # | 위험 요소 | 영향 | 대응 방안 |
|---|-----------|------|-----------|
| 1 | **번역 학습 시간** — 127k×2 샘플의 KoBART 학습이 단일 GPU(M4)에서 길어질 수 있음 | 일정 지연 | 동적 패딩·적정 batch 조정, 필요 시 데이터 부분집합으로 단계적 학습 |
| 2 | **서빙 메모리** — KR-BERT + KoELECTRA + KoBART 동시 로드(16GB 한계) | 추론 실패/지연 | 16GB 내 관리, 필요 시 지연 로드(lazy load) 적용 |
| 3 | **번역 품질 변동** — 짧은 발화 위주라 BLEU 점수 변동이 큼 | 평가 신뢰성 | BLEU와 **chrF 병행 보고**로 보완 |
| 4 | **GPU 경합** — 감정·번역 학습이 동시 실행되면 자원 충돌 | 학습 실패 | GPU 작업 **직렬화**(감정 학습 종료 후 번역 학습) |
