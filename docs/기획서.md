## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **앱 이름 (확정)** | **탐라 (Tamna)** — 제주의 옛 이름에서 따온 가제로, 제주의 정체성과 "모든 것을 품는다"는 의미를 함께 담는다. |
| **한 줄 소개** | 제주 여행·생활에 필요한 모든 것 — 제주어 번역/감정분석, 맛집, 명소·체험, AI 여행경로, 커뮤니티 — 을 한 앱에 담은 **제주 올인원 모바일 웹앱** |
| **형태** | 모바일 우선 반응형 웹앱 (하단 4탭 네비게이션), 추후 PWA/네이티브 래핑 확장 |
| **분류** | 위치기반 서비스(LBS) + NLP(소멸위기언어) + 커뮤니티 + 생성형 AI |
| **대상 학기** | 1학기 번역기 제안 → 2학기 감정분류 → **통합 올인원 앱**(현재) |

### 추진 배경

제주어는 **유네스코가 지정한 소멸위기언어(2010년, "심각한 위기" 등급)** 이며, 표준어 중심 환경과 세대 간 단절로 젊은 세대는 제주어를 이해할 기회가 급격히 줄고 있다. 본 프로젝트는 이 문제의식에서 출발해 1·2학기 동안 **제주어 번역기 제안**과 **제주어 감정분류(F1-Macro 0.84)** 라는 ML 자산을 축적했다.

한편 실제 사용자(여행객·이주자)를 만나며 확인한 것은, 제주에서 필요한 정보(언어·맛집·명소·체험·여행 일정·현지 꿀팁)가 **여러 앱과 블로그·지도·검색에 파편적으로 흩어져 있다**는 점이다. 이에 본 프로젝트는 기존 제주어 ML 성과를 **하나의 탭으로 흡수**하고, 그 위에 맛집 지도·명소 큐레이션·AI 여행경로·커뮤니티를 더해, 제주에 관한 모든 것을 한 곳에서 해결하는 **올인원 앱 "탐라"** 로 범위를 확대한다.

## 2. 문제 정의 및 필요성

### 문제

| # | 문제 | 설명 |
|---|------|------|
| 1 | **정보 분산** | 제주 여행/생활 정보(맛집·명소·체험·교통·꿀팁)가 지도앱, 블로그, 검색, 커뮤니티 등에 흩어져 있어 사용자가 앱을 계속 바꿔 가며 정보를 모아야 한다. |
| 2 | **언어 장벽** | 제주어는 어휘·어미가 표준어와 크게 달라 비제주민·젊은 세대가 현지인의 말, 간판, 메시지를 이해하기 어렵다. |
| 3 | **일정 설계의 어려움** | 가고 싶은 명소를 골라도 동선·일자별 배치를 짜는 것은 수작업이며 시간이 많이 든다. |
| 4 | **현지 정보 부족** | 검증된 현지 꿀팁·질의응답을 모을 공간이 없어, 같은 질문이 분산된 채 반복된다. |
| 5 | **성과의 파편화** | 번역·감정분류 ML 성과가 연구로만 남아, 사용자가 실제로 손에 쥐고 쓰는 제품으로 이어지지 못했다. |

### 필요성

| 관점 | 필요성 |
|------|--------|
| **사용자 편익** | 흩어진 제주 정보를 하나의 앱에서 끊김 없이 — 검색·이해·계획·소통까지 — 해결 |
| **문화 보존** | 소멸위기언어인 제주어의 디지털 접근성을 높여 학습·소통 촉진 |
| **기술 가치** | 저자원 방언 NLP + 위치기반 서비스 + 생성형 AI를 결합한 실증 사례 |
| **활용성** | 관광·콘텐츠·교육·언어연구 등 다방면 응용 기반 마련 |

> 기존 sklearn TF-IDF 베이스라인은 제주어 감정분류에서 F1-Macro **0.30**에 그쳤으나, 딥러닝 전환(KR-BERT 단일 0.76 → 앙상블 0.84)으로 실용 수준을 확보했다. 즉, 본 앱의 탭 1은 **이미 검증된 기술 토대** 위에서 동작한다.

## 3. 목표 및 범위

### 목표

제주 여행·생활에 필요한 핵심 기능을 **4개 탭**으로 통합한 모바일 웹앱 "탐라"를 완성한다. 각 탭은 독립적으로 동작하면서도 사용자 여정 속에서 자연스럽게 이어진다.

### 4탭 기능 상세

| 탭 | 이름 | 핵심 기능 |
|----|------|-----------|
| **1** | **제주어** | 제주어↔표준어 양방향 번역 + 제주어 입력 감정분석(7종, 신뢰도 포함). 기존 ML 자산 활용. |
| **2** | **맛집** | 카카오 지도에 제주 맛집 마커 표시, 마커 클릭 시 상세(이름/카테고리/주소/평점), 지역·카테고리 필터·검색, **길찾기**. |
| **3** | **명소·체험** | 가볼 명소/할 체험을 지도에 큐레이션(자연/문화/체험/포토), 상세·길찾기, 다중 선택, **AI 여행경로 생성**(일자별 동선). |
| **4** | **커뮤니티** | 제주 Q&A·꿀팁·후기 게시/댓글/좋아요. **로그인(Google/Kakao) 필요**, **마이페이지**(내 글·댓글·북마크). |

### MVP 정의 (이번 사이클 목표)

기동 가능한 **4탭 앱 셸**을 기준으로, 키·모델이 없어도 앱이 뜨고 탐색되는 것을 수용 기준으로 한다.

| 탭/요소 | MVP 범위 | 키·모델 미주입 시 동작 |
|---------|----------|------------------------|
| 탭1 제주어 | ML 서비스 연결(번역·감정) | "모델 학습 중" 안내 (503 graceful) |
| 탭2 맛집 | 지도·마커·상세·길찾기 | seed JSON 데이터로 동작, Kakao 키 주입 시 실데이터 |
| 탭3 명소·체험 | 지도·마커·상세·길찾기 + AI경로 라우트 | seed 데이터로 동작, AI경로는 키 주입 시 |
| 탭4 커뮤니티 | 게시판 CRUD(SQLite) + 로그인·마이페이지 | CRUD 동작, 로그인은 OAuth 키 주입 시 |

### 비목표 (YAGNI — MVP에서 명시적으로 제외)

- **결제 / 예약** — 외부 예약·결제 연동 없음
- **실시간 채팅** — 커뮤니티는 비동기 게시/댓글로 충분
- **푸시 알림** — 브라우저 알림 등 미포함
- **네이티브 앱 스토어 배포** — 반응형 웹으로 "어플" 경험 제공, 추후 PWA/네이티브 래핑
- **오프라인 모드** — 온라인 전제

> 위 항목은 모두 *향후 과제*로 남기며, 각 탭이 독립적·모듈식으로 설계되어 확장이 용이하다.

### 성공 지표

| 영역 | 지표 | 목표 |
|------|------|------|
| 감정 분류(탭1) | Test F1-Macro | **0.84** (발표본 재현, 진행 중) |
| 번역(탭1) | BLEU / chrF | 테스트셋 측정·보고, 베이스라인 대비 향상 입증 |
| 앱 셸 | 4탭 기동·탐색 | 키·모델 없이도 앱이 뜨고 모든 탭 탐색 가능 |
| 통합 데모 | end-to-end | 키·모델 주입 시 4탭 전 기능 동작 |

### 페르소나

| 페르소나 | 프로필 | 목표 | 불편(Pain) |
|----------|--------|------|------------|
| **여행객 지민(26)** | 서울 직장인, 3박4일 제주 첫 여행 | 효율적 동선으로 핵심 명소·맛집 둘러보기 | 앱·블로그를 오가며 정보 수집·일정 설계가 번거로움 |
| **이주자 현우(34)** | 1년 차 제주 이주, 카페 운영 | 현지 정보·이웃 소통, 손님의 제주어 이해 | 현지 꿀팁을 물어볼 곳이 마땅치 않음 |
| **학습자 서연(22)** | 제주 출신 대학생, 콘텐츠 제작 관심 | 할머니의 제주어 이해, 제주어 자막 제작 | 제주어 문장 단위 변환·정서 파악 도구 부재 |

### 사용 시나리오 (탭을 넘나드는 여정)

#### 시나리오 1 — 여행객 지민의 첫 제주 여행 (탭3 → 탭2 → 탭1)

> 지민은 3박4일 일정을 짜야 한다.

1. **탭3 (명소·체험)** 에서 가고 싶은 명소(성산일출봉, 카멜리아힐 등)와 체험(승마)을 지도에서 골라 다중 선택한다.
2. **AI 여행경로 생성**을 누르고 "3박4일, 렌터카, 자연·포토 위주"를 입력하니, 일자별 동선이 지도에 폴리라인으로 그려진다.
3. 점심 시간대가 비자, **탭2 (맛집)** 으로 이동해 동선 근처 맛집을 검색하고 길찾기로 연결한다.
4. 현지 식당 간판에서 본 "혼저 옵서예" 문구가 궁금해 **탭1 (제주어)** 에 입력하니 "어서 오세요"로 번역된다.

#### 시나리오 2 — 이주자 현우의 현지 정착 (탭4 → 탭1 → 탭4)

> 현우는 카페 운영 중 궁금한 점을 해결하고 싶다.

1. **탭4 (커뮤니티)** 에 Google 계정으로 로그인하고, "겨울철 제주 카페 손님이 줄 때 팁?"을 질문 글로 올린다.
2. 단골 어르신이 남긴 제주어 메시지의 정서가 궁금해 **탭1**에 입력하니 번역과 함께 "🙂 행복(0.71) — 애정 어린 당부"로 분석된다.
3. 다시 **탭4 마이페이지**에서 내가 쓴 글·받은 댓글을 확인하고, 유용한 꿀팁 글을 북마크한다.

#### 시나리오 3 — 학습자 서연의 콘텐츠 제작 (탭1 → 탭4)

> 서연은 제주어 자막 영상을 만든다.

1. **탭1 (제주어)** 방향 토글을 "표준어→제주어"로 두고 "어서 오세요, 천천히 구경하세요"를 입력해 "혼저 옵서, 놀멍 보멍 갑서" 자막 후보를 얻는다.
2. 역방향(제주어→표준어)으로 다시 검수하고, 감정분석으로 정서 톤을 확인한다.
3. **탭4 (커뮤니티)** 후기 게시판에 제작 과정과 결과물을 공유해 피드백을 받는다.

## 5. 탭별 주요 기능 상세

### 탭 1 — 제주어 (번역 + 감정)

- **양방향 번역:** 입력창 + 방향 토글(제주어→표준어 / 표준어→제주어). 단일 KoBART 모델에 방향 prefix를 부여해 두 방향을 함께 처리.
- **감정 분석(7종):** 제주어 문장의 감정을 **중립 · 슬픔 · 행복 · 분노 · 놀람 · 공포 · 혐오** 7종으로 분류, 신뢰도(확률)를 함께 제시. 이모지·색상 뱃지와 신뢰도 막대로 직관적 표시(단정 대신 분포 제시).
- **결과 카드:** 번역문 + 감정 뱃지·신뢰도를 한 카드에 표시.
- **graceful degradation:** ML 미준비 시 "모델 학습 중" 안내(503).
- **데이터 경로:** Next 서버 라우트 `/api/translate`, `/api/emotion`이 FastAPI ML 서비스(`ML_SERVICE_URL`)를 프록시.

### 탭 2 — 맛집 (지도 · 길찾기)

- **지도 표시:** 카카오 지도에 제주 맛집을 마커로 표시. 데이터 출처는 (a) Kakao Local 키워드/카테고리(FD6 음식점) 검색, (b) 큐레이션 seed JSON(`restaurants.seed.json`) 폴백.
- **상세 정보:** 마커 클릭 시 바텀시트에 이름/카테고리/주소/평점/사진(placeholder).
- **길찾기:** 카카오 지도 길찾기 링크로 연결.
- **탐색:** 지역·카테고리 필터, 키워드 검색.

### 탭 3 — 명소·체험 (지도 + AI 여행경로)

- **큐레이션:** 명소/체험을 카테고리(자연/문화/체험/포토)로 큐레이션(seed JSON `attractions.seed.json`).
- **지도·상세·길찾기:** 마커 표시, 상세 바텀시트, 길찾기 링크.
- **다중 선택(장바구니):** 가고 싶은 장소를 여러 개 담기.
- **AI 여행경로 생성:** 선택 장소 + 사용자 조건(일수 / 이동수단 / 취향)을 서버 라우트 `/api/route`가 **Claude API**로 보내 일자별 동선 일정을 생성. 결과를 지도에 폴리라인으로 표시.

### 탭 4 — 커뮤니티 + 로그인 + 마이페이지

- **게시판:** 카테고리(질문/꿀팁/후기). 목록·상세·작성·댓글·좋아요. **작성/댓글은 로그인 필요.**
- **인증:** NextAuth.js (Google + Kakao provider). 세션은 DB(Prisma adapter)에 저장.
- **마이페이지:** 프로필(이름/이메일/아바타), 내 글·댓글, 좋아요·북마크, 로그아웃.

| 영역 | 선택 | 이유 |
|------|------|------|
| 프론트/앱 | **Next.js 15 (App Router, TypeScript, Tailwind)** | 모바일 반응형, SSR/API 라우트 일체화, 보유 경험 |
| 지도 | **Kakao Maps JS SDK + Kakao Local REST API** | 국내 장소/맛집/길찾기 최적 |
| 길찾기 | Kakao Map 길찾기 링크 / (확장) Kakao Mobility API | 키만으로 즉시 |
| 인증 | **NextAuth.js (Google + Kakao)** | 마이페이지·세션, OAuth 표준 |
| DB | **Prisma + SQLite(dev) / PostgreSQL(prod)** | 커뮤니티·유저, 마이그레이션 용이 |
| AI 여행경로 | **Anthropic Claude API** (서버 라우트) | 선택 명소 기반 일정 생성 |
| ML(번역/감정) | **기존 FastAPI 서비스** (KoBART + 감정 앙상블) | 이미 구현된 Python 자산 재사용 |
| 상태/데이터 | React Server Components + fetch, 필요시 SWR | 단순성 |

### 시스템 구성 (개괄)

```
[모바일 웹 브라우저] ── Next.js 15 (webapp/) ── 하단 4탭 네비
   ├─ 탭1 제주어   → /api/translate · /api/emotion ─proxy→ [FastAPI ML]
   ├─ 탭2 맛집     → Kakao Maps SDK + /api/places  ─→ [Kakao Local]
   ├─ 탭3 명소·체험 → Kakao Maps + /api/route(AI)    ─→ [Claude API]
   ├─ 탭4 커뮤니티  → /api/posts ...                 ─→ [Prisma/DB]
   └─ /api/auth/*  → NextAuth (Google, Kakao)
[FastAPI ML 서비스] 번역(KoBART) + 감정(Dual KR-BERT + KoELECTRA 앙상블)
```

### 설계 원칙

- **서비스 분리:** Next.js(웹앱·커뮤니티·인증·AI라우트)와 Python FastAPI(ML 추론)는 별도 프로세스. Next.js가 `ML_SERVICE_URL`로 ML을 호출.
- **탭 독립성:** 각 탭은 다른 탭 없이도 동작하도록 점진 구현·점진 출시.
- **graceful degradation:** 외부 키(Kakao/OAuth/Claude)·모델이 없으면 seed/mock·안내로 동작. 앱 자체는 키 없이도 기동·탐색 가능. `.env.example`로 키 명세.

- **연구** — 제주어 표현별 감정 분포 관찰, 방언 NLP 후속 연구의 베이스.
- **확장 기반** — PWA/네이티브, 결제·예약, 실시간 기능 등 향후 확장의 토대.

## 8. 추진 일정

### 전체 단계

| 단계 | 기간 | 내용 | 산출물 |
|------|------|------|--------|
| **1단계** | 1학기 | 제주어 **번역기 제안** 및 데이터/접근 정의 | 번역기 제안서, 병렬 코퍼스 정리 |
| **2단계** | 2학기 | 제주어 **감정분류** 모델 개발 (KR-BERT → 앙상블) | F1-Macro 0.84 모델, 감정분류 보고서 |
| **3단계** | 통합기 (현재) | 번역 학습 + 두 모델을 흡수한 **올인원 앱 "탐라"** 구축 | 4탭 웹앱, 기획서·개발설계서 |

### 통합기 세부 작업 순서

| 순서 | 작업 | GPU | 비고 |
|------|------|-----|------|
| 1 | 문서화(기획서·개발설계서) + Next.js 앱 스캐폴드(`webapp/`) | 비-GPU | 병렬 진행 |
| 2 | 감정 모델 학습 완료 → F1 0.84 검증 | GPU | 진행 중 (직렬) |
| 3 | 번역 KoBART 학습 → BLEU/chrF 평가 | GPU | 감정 학습 종료 후 (직렬) |
| 4 | 탭별 기능 점진 구현(맵·커뮤니티·AI경로) + ML 연결 | 비-GPU | 탭 독립 구현 |
| 5 | end-to-end 데모 검증, 리포트·문서 갱신, 최종 발표 준비 | 비-GPU | |

> GPU 작업(감정·번역 학습)은 단일 GPU 환경을 고려해 **직렬 실행**하며, 앱 개발(비-GPU)과 병렬로 진행한다.
