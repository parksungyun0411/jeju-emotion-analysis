🔸 1. Naive Bayes (나이브 베이즈)
------------------------------------------------------------

📋 기본 원리:
- 베이즈 정리를 기반으로 한 확률적 분류 모델
- 각 특성(단어)이 독립적이라고 가정 (Naive 가정)
- P(감정|텍스트) = P(텍스트|감정) × P(감정) / P(텍스트)

🔧 작동 방식:
1. 각 감정 클래스별로 단어 출현 확률 계산
2. 새로운 텍스트에서 단어들이 나타날 확률을 각 감정별로 계산
3. 가장 높은 확률을 가진 감정으로 분류

✅ 장점:
- 학습 속도가 매우 빠름
- 작은 데이터셋에서도 잘 작동
- 해석이 용이함
- 과적합에 강함
- 균형 데이터에서 좋은 성능 (30.77% - 최고 성능)

❌ 단점:
- 특성 독립성 가정이 현실적이지 않음
- 희소한 단어에 민감함
- 불균형 데이터에서 성능 저하 (50.39%)

📊 우리 실험 결과:
- 최종데이터.xlsx: 50.39% (3위)
- 최종데이터_균형.xlsx: 30.77% (1위) ⭐
- 균형 데이터에서 최고 성능 달성

🔸 2. Logistic Regression (로지스틱 회귀)
------------------------------------------------------------

📋 기본 원리:
- 선형 분류 모델의 일종
- 시그모이드 함수를 사용하여 확률로 변환
- P(감정) = 1 / (1 + e^(-z)), z = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ

🔧 작동 방식:
1. 각 단어(특성)에 가중치 부여
2. 가중합을 계산하여 선형 결합 생성
3. 시그모이드 함수로 확률 변환
4. 가장 높은 확률의 감정으로 분류

✅ 장점:
- 해석이 매우 용이함 (가중치로 중요도 파악)
- 과적합에 상대적으로 강함
- 안정적인 성능
- 빠른 학습 속도
- 불균형 데이터에서도 안정적 (52.33%)

❌ 단점:
- 선형 관계만 모델링 가능
- 복잡한 패턴 학습 한계
- 특성 간 상호작용 고려 어려움

📊 우리 실험 결과:
- 최종데이터.xlsx: 52.33% (2위)
- 최종데이터_균형.xlsx: 30.19% (2위)
- 두 데이터셋 모두에서 안정적인 2위 성능

🔸 3. SVM (Support Vector Machine, 서포트 벡터 머신)
------------------------------------------------------------

📋 기본 원리:
- 최대 마진 분류기 (Maximum Margin Classifier)
- 클래스 간 경계를 최대한 넓게 만드는 초평면 찾기
- 서포트 벡터(경계에 가장 가까운 데이터)를 기반으로 분류

🔧 작동 방식:
1. 고차원 공간에서 데이터를 매핑
2. 클래스 간 마진을 최대화하는 초평면 찾기
3. 커널 트릭을 사용하여 비선형 분류 가능
4. 우리 실험에서는 linear 커널 사용

✅ 장점:
- 고차원 데이터에서 효과적
- 메모리 효율적 (서포트 벡터만 저장)
- 과적합에 강함
- 불균형 데이터에서 최고 성능 (52.54%)

❌ 단점:
- 대용량 데이터에서 학습 속도 느림
- 하이퍼파라미터 튜닝 중요
- 해석이 어려움
- 균형 데이터에서 성능 저하 (29.50%)

📊 우리 실험 결과:
- 최종데이터.xlsx: 52.54% (1위) ⭐
- 최종데이터_균형.xlsx: 29.50% (3위)
- 불균형 데이터에서 최고 성능 달성

🔸 4. Random Forest (랜덤 포레스트)
------------------------------------------------------------

📋 기본 원리:
- 앙상블 학습 방법
- 여러 개의 의사결정나무를 조합하여 예측
- 배깅(Bagging)과 랜덤 특성 선택을 결합

🔧 작동 방식:
1. 부트스트랩 샘플링으로 여러 데이터셋 생성
2. 각 데이터셋으로 의사결정나무 학습
3. 각 나무마다 랜덤하게 특성 선택
4. 모든 나무의 예측을 투표하여 최종 결정

✅ 장점:
- 과적합에 매우 강함
- 특성 중요도 제공
- 결측값 처리 가능
- 다양한 데이터 타입 지원
- 안정적인 성능

❌ 단점:
- 학습 시간이 오래 걸림
- 메모리 사용량 많음
- 해석이 복잡함
- 두 데이터셋 모두에서 최저 성능 (49.64%, 26.68%)

📊 우리 실험 결과:
- 최종데이터.xlsx: 49.64% (4위)
- 최종데이터_균형.xlsx: 26.68% (4위)
- 두 데이터셋 모두에서 최저 성능

1. **SVM**
   - 불균형 데이터에서 최고 성능
   - 고차원 특성 공간에서 효과적
   - 마진 최대화로 일반화 성능 우수

3. **Naive Bayes**
   - 균형 데이터에서 최고 성능
   - 빠른 학습 속도
   - 작은 데이터셋에서 효과적

4. **Random Forest**
   - 두 데이터셋 모두에서 최저 성능
   - 텍스트 분류에는 부적합할 수 있음

   - 전체 정확도는 높지만 F1-Macro는 낮음

2. **균형 데이터 (최종데이터_균형.xlsx)**
   - Naive Bayes가 가장 효과적
   - 각 클래스가 동일한 비중으로 학습
   - F1-Macro와 F1-Weighted가 유사

📊 실험 설계 및 방법론
============================================================

🔧 전처리 과정:
1. 텍스트 정제 (특수문자 제거, 공백 정리)
2. TF-IDF 벡터화 (1-3 gram, 30,000 특성)
3. 80:20 훈련/테스트 분할
4. 계층적 샘플링 (stratified sampling)

🔧 평가 지표:
- Accuracy: 전체 정확도
- F1-Macro: 클래스별 F1의 평균 (불균형 고려)
- F1-Weighted: 클래스 크기에 따른 가중 평균

🔧 하이퍼파라미터:
- TF-IDF: max_features=30,000, ngram_range=(1,3)
- SVM: kernel='linear', probability=True
- Random Forest: n_estimators=100
- Logistic Regression: max_iter=1000, C=1.0
- Naive Bayes: alpha=1.0
