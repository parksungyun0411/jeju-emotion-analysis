# 제주 올인원 앱 — 개발설계서 (v2)

> 전면 재작성 2026-05-25. 본 문서는 마스터 설계 v2
> (`docs/superpowers/specs/2026-05-25-jeju-allinone-app-design.md`)를 구현 관점에서
> 정교화한 **개발설계서**다. 범위는 "제주어 챗봇"에서 **제주도에 관한 모든 것**을 담는
> 4탭 모바일 웹앱(가제 `탐라/제주올`)으로 확대되었다.
>
> 식별자(클래스명·DataFrame 컬럼명·엔드포인트·환경변수)는 실제 코드
> (`src/jeju_kr_bert.py`, `src/translation/train_translation.py`,
> `app/main.py`, `app/services/*.py`)에서 그대로 가져왔다.
>
> 핵심 아키텍처 결정: **Next.js(앱·API라우트·인증·커뮤니티)** 와
> **Python FastAPI(ML 추론)** 를 별도 프로세스로 분리하고, 모든 외부 의존
> (Kakao/OAuth/Claude/ML/DB)은 **키·모델 부재 시 graceful degradation** 하도록 설계한다.

## 1. 시스템 아키텍처

3-tier 구성. **브라우저**(프레젠테이션) ↔ **Next.js 15**(앱 셸·API 라우트·인증·커뮤니티,
BFF 역할) ↔ **백엔드 의존 4종**(FastAPI ML / Kakao API / Claude API / DB). 브라우저는
Kakao Maps JS SDK 한 곳에만 직접 접근(공개 JS 키)하고, 그 외 모든 비밀키 호출은 Next.js
서버 라우트가 프록시한다.

```
┌──────────────────────────────────────────────────────────────────────────┐
│  모바일 웹 브라우저  (반응형, 하단 4탭 네비게이션)                          │
│                                                                            │
│   Kakao Maps JS SDK ───(NEXT_PUBLIC_KAKAO_MAP_JS_KEY, 공개)──┐             │
└───────────────┬───────────────────────────────────────────  │  ───────────┘
                │ HTTPS (RSC payload / fetch / form action)    │
                ▼                                              ▼
┌──────────────────────────────────────────────────────┐  ┌──────────────────┐
│  Next.js 15  (App Router, webapp/)                     │  │  Kakao Maps 타일  │
│  ──────────────────────────────────────────────────   │  │  (브라우저 직결)  │
│  페이지(RSC/Client)                                    │  └──────────────────┘
│   ├─ /         (탭1 제주어)  → /api/translate /api/emotion
│   ├─ /food     (탭2 맛집)    → /api/places + Kakao JS SDK 지도
│   ├─ /spots    (탭3 명소체험)→ /api/places /api/route + 지도
│   ├─ /community(탭4 커뮤니티)→ /api/posts ... (로그인 가드)
│   └─ /mypage                 → 세션 기반
│                                                        │
│  서버 라우트 (app/api/*  =  BFF·프록시·비밀키 보관)    │
│   ┌─ /api/translate ─┐                                 │
│   ┌─ /api/emotion   ─┤── proxy(ML_SERVICE_URL) ───────────────┐
│   ┌─ /api/places ────── Kakao Local REST(KAKAO_REST_API_KEY)──┼──┐
│   ┌─ /api/route ──────── Anthropic Claude(ANTHROPIC_API_KEY)──┼──┼──┐
│   ┌─ /api/posts|comments|like|bookmark ─ Prisma ──────────────┼──┼──┼──┐
│   └─ /api/auth/[...nextauth] ─ NextAuth(Google, Kakao) ───────┘  │  │  │
└──────────────────────────────────────────────────────┘         │  │  │
        │                          │                              │  │  │
        ▼                          ▼                              ▼  ▼  ▼
┌──────────────────┐   ┌──────────────────┐   ┌─────────┐  ┌─────────┐ ┌──────────┐
│ FastAPI ML 서비스│   │ Kakao Local REST │   │ Claude  │  │ OAuth   │ │ DB       │
│ (app/main.py,    │   │ (장소/맛집 검색) │   │  API    │  │ Google  │ │ Prisma   │
│  127.0.0.1:8000) │   └──────────────────┘   │(여행경로)│  │ /Kakao  │ │ SQLite→PG│
│  POST /api/translate                        └─────────┘  └─────────┘ └──────────┘
│  POST /api/emotion │
│  POST /api/chat    │   ┌─────────────────────────────────────────────────────┐
│  GET  /api/health  │   │ ML 내부:                                            │
│        │           │   │  번역  = KoBART 양방향(방향 prefix) seq2seq         │
│        └───────────┼──▶│  감정  = Dual KR-BERT ⊕ KoELECTRA 앙상블            │
│                    │   │          p_ens = α·p_dual + (1-α)·p_single          │
└────────────────────┘   └─────────────────────────────────────────────────────┘
```

**프로세스 분리:** Next.js(포트 3000)와 FastAPI(포트 8000)는 독립 기동한다. Next.js는
`ML_SERVICE_URL`(기본 `http://127.0.0.1:8000`)로 ML 서비스를 호출한다.

**graceful degradation 경계:** 위 5개 백엔드 의존 각각은 부재 시 해당 기능만 비활성화되고
앱 셸·다른 탭은 정상 동작한다(§3, §8 참조). 키·모델 없이도 앱이 뜨고 탐색되는 것이 수용 기준.

## 3. 탭별 상세 설계

각 탭은 독립적으로 동작한다(다른 탭 없이도 기동). 공통 셸: 하단 고정 4탭 네비게이션
(`components/BottomNav`), 모바일 우선 레이아웃(`app/layout.tsx`).

### 탭 1 — 제주어 (번역 + 감정)

| 항목 | 내용 |
|------|------|
| 라우트 | `/` (`app/page.tsx`) |
| 컴포넌트 | `JeoTab`(컨테이너), `TextInputCard`(입력창+방향 토글 j2s/s2j), `TranslationResultCard`(번역문), `EmotionBadge`(감정 라벨·신뢰도), `LoadingState` |
| 데이터 흐름 | 입력 → `POST /api/translate {text, direction}` 및 (제주어 입력 시) `POST /api/emotion {text}` → Next 서버 라우트가 `ML_SERVICE_URL` FastAPI로 프록시 → 결과 카드 렌더 |
| 외부 의존 | FastAPI ML (`/api/translate`, `/api/emotion`) |
| graceful degradation | ML 모델 미준비 시 FastAPI가 **503 + 안내 메시지** 반환(`TranslationUnavailableError`/`EmotionUnavailableError`). 라우트는 503을 그대로 전달하고 UI는 "모델 학습 중입니다" 배너 표시. 감정만 미준비면 번역 결과는 그대로 보여줌(부분 degradation). |

감정 라벨(7종, `EMOTION_LABELS`와 1:1):
`0 중립 / 1 슬픔 / 2 행복 / 3 분노 / 4 놀람 / 5 공포 / 6 혐오`.

### 탭 2 — 맛집 (지도)

| 항목 | 내용 |
|------|------|
| 라우트 | `/food` (`app/food/page.tsx`) |
| 컴포넌트 | `KakaoMap`(JS SDK 래퍼, 마커 렌더), `RestaurantMarker`, `PlaceBottomSheet`(이름/카테고리/주소/평점/사진 placeholder + 길찾기 버튼), `FilterBar`(지역/카테고리), `SearchBox` |
| 데이터 흐름 | 지도 영역/검색어 → `GET /api/places?query=&category=FD6&lat=&lng=` → Kakao Local 프록시 → 마커. 마커 클릭 → 바텀시트 상세 → "길찾기" → Kakao Map 길찾기 링크(`https://map.kakao.com/link/to/...`) |
| 데이터 출처 | (a) Kakao Local `keyword`/`category`(FD6 음식점), (b) seed 폴백 `webapp/data/restaurants.seed.json` |
| 외부 의존 | Kakao Maps JS SDK(브라우저, `NEXT_PUBLIC_KAKAO_MAP_JS_KEY`), Kakao Local REST(`KAKAO_REST_API_KEY`, 서버) |
| graceful degradation | JS 키 없음 → 지도 대신 "지도 키 미설정" placeholder + 리스트 뷰. REST 키 없음 → `/api/places`가 seed JSON 반환(소수 큐레이션 맛집). |

### 탭 3 — 명소·체험 (지도 + AI 여행경로)

| 항목 | 내용 |
|------|------|
| 라우트 | `/spots` (`app/spots/page.tsx`) |
| 컴포넌트 | `KakaoMap`(공유), `SpotMarker`, `PlaceBottomSheet`(상세+길찾기), `CategoryTabs`(자연/문화/체험/포토), `SpotCart`(다중 선택 장바구니), `RoutePlannerSheet`(일수/이동수단/취향 입력), `RoutePolyline`(생성 동선 표시) |
| 데이터 흐름 | 명소 큐레이션(`webapp/data/attractions.seed.json`) + (확장) `GET /api/places` → 마커. 장바구니에 선택 → `RoutePlannerSheet`에서 조건 입력 → `POST /api/route {spots[], days, transport, prefs}` → Claude API가 일자별 동선 일정 생성(JSON) → 지도에 폴리라인 + 일정 카드 |

| 항목 | 내용 |
|------|------|
| 라우트 | `/community`(목록), `/community/[id]`(상세), `/community/new`(작성), `/mypage` |

## 4. ML 서비스 설계

FastAPI(`app/main.py`)가 번역·감정을 REST로 노출한다. 두 서비스는 **지연 로드**:
앱 기동 시 인스턴스만 생성하고 모델 가중치는 첫 호출 시 로드한다. 아티팩트가 없으면
`available == False`로 두고 앱은 정상 기동, 호출 시 503을 반환한다.

### 4.1 FastAPI 엔드포인트 (실제 `app/main.py`)

| 메서드·경로 | 요청 모델 | 응답 모델 | 비고 |
|---|---|---|---|
| `GET /api/health` | — | `HealthResponse{status, services}` | `translation_service.status()`, `emotion_service.status()` 집계 |
| `POST /api/translate` | `TranslateRequest{text, direction}` | `TranslateResponse{translation}` | `direction` 기본 `"j2s"`; 미가용 503, 잘못된 입력 400 |
| `POST /api/emotion` | `EmotionRequest{text}` | `EmotionResponse{label, label_id, scores}` | `scores`는 7감정명→확률 dict |
| `POST /api/chat` | `ChatRequest{text, direction}` | `ChatResponse{translation, emotion?}` | j2s면 번역+감정, s2j면 번역만 |

`direction` 값은 `"j2s"`(제주어→표준어), `"s2j"`(표준어→제주어). (Next.js 탭1은
`/api/translate`,`/api/emotion`만 프록시; `/api/chat`은 ML 서비스 직접 데모용.)

### 4.2 번역 — KoBART 양방향 prefix (`src/translation/train_translation.py`)

- **베이스 모델:** `MODEL_NAME = "gogamza/kobart-base-v2"` (`AutoModelForSeq2SeqLM`).
- **양방향 단일 모델 + 방향 prefix** — 한 모델로 두 방향 학습. `PREFIX`:
  - `j2s` = 제주어→표준어: 입력 `"표준어로: " + 제주어` → 타깃 `표준어`
  - `s2j` = 표준어→제주어: 입력 `"제주어로: " + 표준어` → 타깃 `제주어`
- `--direction both`(기본): 각 병렬쌍을 j2s·s2j 두 샘플로 확장(데이터 2배). `build_pairs()` 가
  `(sources, targets, sample_dirs)` 생성.
- **컬럼:** `JEJU_COL = "제주어 문장"`, `STD_COL = "표준어 문장"`.
- **분할:** `split_811()` — 8:1:1 Train/Val/Test (감정과 분할 정합성 불필요).
- **데이터셋/패딩:** `TranslationDataset`(`__init__` 1회 토큰화) + `DataCollatorForSeq2Seq`
  (동적 패딩, `label_pad_token_id=-100`).
- **HP:** `lr=5e-5, batch=32, epochs=5, max_length=64, weight_decay=0.01, warmup_ratio=0.10,
  grad_clip=1.0, num_beams=4`.
- **평가:** beam search 생성(`num_beams=4`) 후 `sacrebleu`로 **BLEU + chrF**, 방향별 분리 보고
  (`evaluate()` → `{j2s, s2j, overall}`). 비용 절감 위해 val `eval_samples=1500`,
  test `test_samples=4000` 부분집합.
- **산출물:** `results_translation/kobart_jeju/`(save_pretrained) + `translation_report.txt`.
- **추론 서비스:** `app/services/translation_service.py`의 `TranslationService`가
  `src/translation/translate.py`의 `Translator(artifacts_dir).translate(text, direction)`를 재사용.
  미가용 시 `TranslationUnavailableError`.

### 4.3 감정 — Dual KR-BERT + KoELECTRA 앙상블 (`src/jeju_kr_bert.py`)

발표본 단계별 성능(목표 최종 **Test F1 = 0.8404**):

| Step | 구성 | Test F1 | CLI `--step` |
|---|---|---|---|
| 1 | KR-BERT 베이스라인(전처리 없음) | 0.4733 | `baseline` |
| 2 | + 데이터 밸런싱(오버샘플링) | 0.5833 | `balanced` |
| 3 | + Dual Stream(KR-BERT 듀얼 + 게이팅) | 0.6158 | `dual` |
| 4 | + 토크나이저 최적화(제주어 토큰) | 0.7184 | `tokenizer` |
| 5 | + DAPT(제주어 MLM 재학습) | 0.7456 | `dapt` |
| 6 | + Hard Mining(loss 상위 20% 가중) | 0.8003 | `hard-mining` |
| 7 | + **앙상블**(KoELECTRA + Dual KR-BERT) | **0.8404** | `ensemble` |
| — | KoELECTRA single (Branch 1) | — | `koelectra` |

- **라벨:** `EMOTION_LABELS = {0:"중립",1:"슬픔",2:"행복",3:"분노",4:"놀람",5:"공포",6:"혐오"}`,
  `NUM_LABELS = 7`.
- **컬럼:** `'제주어 문장'`, `'표준어 문장'`, `'감정번호'`(라벨). 데이터
  `merged_jeju_balanced_sorted.xlsx`(127,324행).
- **분할:** `stratified_split_712(df, label_col='감정번호')` — Stratified 7:1:2.
  single/dual이 **같은 행 집합**을 쓰도록 두 언어 컬럼 모두 NA 제거(앙상블 test set 정합).
- **HP(`DEFAULT_HP`):** `lr=3e-5, batch=64, epochs=7, max_length=64, dropout=0.2,
  weight_decay=0.01, warmup_ratio=0.20, label_smoothing=0.05, grad_clip=1.0, seed=42`.

**Dual-Gated KR-BERT (`DualGatedClassifier`, Step 3):**
공유 인코더 1개로 제주어/표준어 각각 `[CLS]` 추출 후 게이팅.
```
h_j = encode_cls(jeju_ids, jeju_mask)      # 제주어 [CLS]
h_s = encode_cls(std_ids,  std_mask)       # 표준어 [CLS]
g        = σ(W_g · [h_j ; h_s] + b_g)      # self.gate: Linear(hidden*2, hidden)
h_final  = g · h_j + (1 - g) · h_s
logits   = classifier(dropout(h_final))    # self.classifier: Linear(hidden, 7)
loss     = CrossEntropy(logits, labels, label_smoothing=0.05)
```
데이터셋 `DualTextDataset` + `make_dual_collator`(제주어/표준어 각각 동적 패딩).
state_dict 저장: `results_kr_bert/step3_dual_state.pt`.

**KoELECTRA single (Branch 1):** `AutoModelForSequenceClassification`
(`monologg/koelectra-base-v3-discriminator`), `SingleTextDataset` + `DataCollatorWithPadding`.
디렉토리 저장: `results_kr_bert/branch1_koelectra_model/`.

**Hard Mining(Step 6):** 1 epoch 정상 학습 후, `shuffle=False` 로더로 샘플별 loss 계산
(`_compute_per_sample_loss`), 상위 20% loss 샘플 가중치 `alpha=2.0`
(`_hard_mining_weights`), `WeightedRandomSampler`로 오버샘플.

**앙상블(`run_step_ensemble`, Step 7):**
```
p_ens = α · p_dual + (1 - α) · p_single
```
- `p_dual`: Dual KR-BERT softmax 확률, `p_single`: KoELECTRA softmax 확률.
- 두 단계가 저장한 `*_probs.npz`(probs, labels)를 로드, **labels 동일성 검증**
  (`assert np.array_equal` — seed 고정 stratified split이 일치해야 함).
- α는 `alpha_grid = np.linspace(0, 1, 21)`에서 **F1-macro 최대**가 되는 값 탐색.
- 산출물: `step7_ensemble_report.txt`(`Extra: alpha: <값>`).

**추론 서비스(`app/services/emotion_service.py`, `EmotionService`):**
- 가용 조건: `step3_dual_state.pt` 존재 **and** `branch1_koelectra_model/` 디렉토리 존재
  (둘 다 필요; 한쪽만 있으면 unavailable).
- α는 `step7_ensemble_report.txt`에서 정규식 파싱(`_parse_alpha`), 없으면 **0.5**.
- 추론 시 표준어 번역이 없으므로 Dual 모델에 **동일 텍스트를 jeju/std 양쪽 입력**
  (게이트가 자체 조정). `p_ens` argmax → `{label, label_id, scores}`.
- 미가용 시 `EmotionUnavailableError`. 무거운 import(`torch`/`transformers`)는 지연.

모든 라우트는 Next.js App Router(`webapp/app/api/.../route.ts`). 비밀키 호출은 서버에서만.

| 메서드·경로 | 요청 | 응답(예) | 비고 |
|---|---|---|---|
| `POST /api/translate` | `{ "text":"...", "direction":"j2s" }` | `{ "translation":"..." }` | ML 프록시. 503 시 `{error:"모델 학습 중"}` |
| `POST /api/emotion` | `{ "text":"..." }` | `{ "label":"행복","label_id":2,"scores":{"중립":0.05,...} }` | ML 프록시(7감정 dict) |
| `GET /api/places` | `?query=흑돼지&category=FD6&lat=33.49&lng=126.53` | `{ "items":[{"id","name","category","address","lat","lng","phone","placeUrl"}], "source":"kakao"\|"seed" }` | Kakao Local 프록시 + seed 폴백 |
| `POST /api/route` | `{ "spots":[{"name","lat","lng"}], "days":2, "transport":"car", "prefs":"자연 위주" }` | `{ "itinerary":[{"day":1,"stops":[{"name","order","note"}]}], "source":"claude"\|"rule" }` | Claude 일정 생성, 키 없으면 규칙 기반 |

| `POST /api/posts` | `{ "category","title","body" }` | `{ "id","title",... }` | **로그인 필요**(401 가드) |
| `GET /api/posts/[id]` | — | `{ "id","title","body","author","createdAt","likeCount","comments":[...] }` | 공개 |
| `POST /api/posts/[id]/comments` | `{ "body":"..." }` | `{ "id","body","author","createdAt" }` | **로그인 필요** |
| `POST /api/posts/[id]/like` | — (토글) | `{ "liked":true,"likeCount":12 }` | **로그인 필요** |
| `POST /api/bookmarks` | `{ "targetType":"restaurant","targetId":"..." }` | `{ "bookmarked":true }` | **로그인 필요** |
| `GET /api/mypage` | — | `{ "user":{...}, "posts":[...], "comments":[...], "bookmarks":[...] }` | **로그인 필요** |
| `GET/POST /api/auth/[...nextauth]` | NextAuth 표준 | 세션/콜백 | Google, Kakao |

**오류 규약:** 입력 오류 400, 비로그인 보호 라우트 401, ML/외부 미가용 503,
서버 오류 500. 본문은 `{ "error": "<메시지>" }`. ML 503은 FastAPI의 `detail`을 그대로 전달.

**프록시 예 (`/api/translate`):**
```ts
// webapp/app/api/translate/route.ts
export async function POST(req: Request) {
  const body = await req.json();                 // {text, direction}
  const base = process.env.ML_SERVICE_URL ?? "http://127.0.0.1:8000";
  const r = await fetch(`${base}/api/translate`, {
    method: "POST", headers: { "content-type": "application/json" },
    body: JSON.stringify({ text: body.text, direction: body.direction ?? "j2s" }),
  });
  if (r.status === 503) return Response.json({ error: "모델 학습 중입니다." }, { status: 503 });
  return Response.json(await r.json(), { status: r.status });
}
```

## 7. 인증·인가 설계

- **프로바이더:** NextAuth.js `GoogleProvider` + `KakaoProvider`
  (`webapp/app/api/auth/[...nextauth]/route.ts` + `webapp/lib/auth.ts`).
- **세션 전략:** **Database 세션**(`PrismaAdapter`, `Session`/`Account` 모델). `getServerSession(authOptions)`로
  서버 라우트·RSC에서 세션 조회. 세션에 `userId` 노출(`callbacks.session`).
- **로그인 가드(인가):**
  - 보호 서버 라우트(`POST /api/posts`, `/comments`, `/like`, `/bookmarks`, `GET /api/mypage`):
    핸들러 진입부에서 `getServerSession` → 없으면 **401**.
  - 보호 페이지(`/community/new`, `/mypage`): RSC에서 세션 없으면 `redirect("/api/auth/signin")`.
  - 읽기(목록·상세)는 비로그인 허용.
- **마이페이지(`/mypage`):** 세션 `userId`로 프로필(`name/email/image`), 내 글(`Post where authorId`),
  내 댓글(`Comment`), 좋아요/북마크(`Like`/`Bookmark`) 조회. 로그아웃(`signOut`).
- **graceful degradation:** OAuth 키 미설정 시 해당 프로바이더 버튼 숨김. 둘 다 없으면 커뮤니티는
  읽기 전용 + "로그인 필요" 안내(앱 기동·열람은 정상).

### 10.3 스모크 시나리오 (수동/통합)

3. **탭1 ML 가용:** 아티팩트 배치 후 `GET /api/health`의 `services.emotion.available==true`,
   `services.translation.available==true` → 번역문 + 감정 뱃지 표시.
4. **탭2 맛집:** Kakao 키 없음 → seed 리스트 표시. 키 주입 → 지도·마커·길찾기 링크.
5. **탭3 AI경로:** Claude 키 없음 → 규칙 기반 일정. 키 주입 → Claude 일자별 일정 + 폴리라인.
6. **탭4 커뮤니티:** OAuth 키 없음 → 읽기 전용. 키 주입 → 로그인·작성·댓글·좋아요·마이페이지.

### 10.4 자동 테스트(권장)

- **API 라우트 단위:** `/api/places` seed 폴백, `/api/route` 규칙 폴백, 로그인 가드 401 (Vitest).
- **ML 서비스 단위:** `EmotionService.available`/`status()`, `TranslationService.available`,
  미가용 시 `*UnavailableError` (pytest, 아티팩트 부재 케이스).
- **감정 앙상블 정합:** `run_step_ensemble`의 labels 동일성 `assert`(seed 고정 분할) — 회귀 방지.
