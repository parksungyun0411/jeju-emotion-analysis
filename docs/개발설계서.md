### 1.2 데이터 흐름 (POST /api/chat, 제주어 입력 기준)

```
사용자 입력(제주어)
   │
   ├──▶ TranslationService.translate(text, "j2s")
   │        prefix "표준어로: " + text → KoBART.generate(beam) → 표준어 번역
   │
   └──▶ EmotionService.predict(text)
            제주어 text → KoELECTRA 토크나이즈 → p_single
            제주어 text + (번역 표준어 또는 동일 text) → Dual-Gated KR-BERT → p_dual
            p_ens = α·p_dual + (1-α)·p_single → label, label_id, scores[7]
   │
   ▼
응답 JSON { translation, emotion:{ label, label_id, scores } }
```

> 비고: Dual-Gated KR-BERT는 학습 시 `제주어 문장`과 `표준어 문장` 쌍을 모두 입력으로 받는다
> (§2.A.3). 서빙 단계에서 표준어 쌍이 없으면 번역 결과를 표준어 입력으로 재사용하거나
> 제주어 문장을 양쪽에 넣는 폴백을 둔다(구현 시 `emotion_service` 옵션).

## 2. 서브시스템별 상세 설계

### 2.A 감정분류 (기존, `src/jeju_kr_bert.py`)

발표본 7단계 파이프라인을 그대로 구현한다. 최종 산출물은 Dual-Gated KR-BERT와
KoELECTRA의 가중 앙상블이며, 목표는 Test F1-Macro **0.84**다.

#### 2.A.1 데이터

| 항목 | 값 |
|------|------|
| 파일 | `merged_jeju_balanced_sorted.xlsx` (127,324행, 7클래스 거의 균형) |
| 사용 컬럼 | `제주어 문장`, `표준어 문장`, `감정번호` (보조: `gpt감정`) |
| 클래스(`EMOTION_LABELS`) | `0 중립, 1 슬픔, 2 행복, 3 분노, 4 놀람, 5 공포, 6 혐오` (`NUM_LABELS=7`) |
| 결측 처리 | `dropna(subset=['제주어 문장','표준어 문장','감정번호'])` — single/dual 단계가 **동일 행 집합** 사용 |
| 분할 | `stratified_split_712()` — seed=42 고정 stratified **7:1:2** (Train/Val/Test) |
| 분할 규모(발표본) | Train 89,118 / Val 12,732 / Test 25,463 |

분할 구현(실제 코드):

```python
def stratified_split_712(df, label_col='감정번호', seed=42):
    train_val, test = train_test_split(df, test_size=0.2, random_state=seed,
                                        stratify=df[label_col])
    train, val = train_test_split(train_val, test_size=1/8, random_state=seed,
                                  stratify=train_val[label_col])   # 7:1
    return train.reset_index(drop=True), val.reset_index(drop=True), test.reset_index(drop=True)
```

> seed=42 고정으로 모든 단계가 **동일한 test set**을 사용하므로, 앙상블 단계에서
> `p_dual`과 `p_single`의 라벨 정합성(`assert np.array_equal(labels_d, labels_s)`)이 보장된다.

#### 2.A.2 7단계 파이프라인과 단계별 성능

| 단계 | `--step` | 기법 | 함수 | 발표본 F1 |
|------|----------|------|------|----------:|
| 1 | `baseline` | KR-BERT 베이스라인(`step1_baseline`) | `run_step_single` | 0.4733 |
| 2 | `balanced` | 데이터 밸런싱(`step2_balanced`, 본 데이터는 이미 균형) | `run_step_single` | 0.5833 |
| 3 | `dual` | Dual Stream(KR-BERT 듀얼+게이팅, `step3_dual`) | `run_step_dual` | 0.6158 |
| 4 | `tokenizer` | 토크나이저 최적화(제주어 토큰 추가, `step4_tokenizer`) | `run_step_single` | 0.7184 |
| 5 | `dapt` | DAPT(제주어 MLM 재학습, `step5_dapt`) | `run_step_single` | 0.7456 |
| 6 | `hard-mining` | Hard Mining(loss 상위 20% 가중, `step6_hard_mining`) | `run_step_single` | 0.8003 |
| - | `koelectra` | Branch 1 KoELECTRA(`branch1_koelectra`) | `run_step_single` | - |
| 7 | `ensemble` | 앙상블(KoELECTRA + Dual KR-BERT, `step7_ensemble`) | `run_step_ensemble` | **0.8404** |

> 재현 검증 중 단일 KR-BERT 베이스라인(epoch 2)에서 이미 Test F1-Macro 0.7593을 기록했다.
> 사용 데이터가 이미 밸런싱·정제된 상태라 발표본의 raw baseline(0.4733)보다 출발점이 높다
> (`docs/bert_improvement_report.md` §3 참조).

#### 2.A.3 Dual-Gated KR-BERT 구조 (`DualGatedClassifier`)

발표본 슬라이드 14~15. **Shared encoder(하나의 BERT)**로 제주어/표준어를 각각 인코딩하고
게이트로 융합한다.

```
제주어 문장 ─┐
            ├─▶ Shared KR-BERT (snunlp/KR-BERT-char16424) ─▶ [CLS]_jeju = h_j
표준어 문장 ─┘                                            ─▶ [CLS]_std  = h_s

g       = σ( W_g · [h_j ; h_s] + b_g )      # gate = nn.Linear(hidden*2, hidden)
h_final = g · h_j + (1 - g) · h_s           # element-wise 게이팅
logits  = classifier( dropout(h_final) )    # classifier = nn.Linear(hidden, 7)
loss    = cross_entropy(logits, labels, label_smoothing=0.05)
```

실제 forward 구현:

```python
def forward(self, jeju_ids, jeju_mask, std_ids, std_mask, labels=None, label_smoothing=0.05):
    h_j = self.encode_cls(jeju_ids, jeju_mask)        # last_hidden_state[:, 0, :]
    h_s = self.encode_cls(std_ids, std_mask)
    g = torch.sigmoid(self.gate(torch.cat([h_j, h_s], dim=-1)))
    h_final = g * h_j + (1 - g) * h_s
    logits = self.classifier(self.dropout(h_final))
    loss = F.cross_entropy(logits, labels, label_smoothing=label_smoothing) if labels is not None else None
    return loss, logits, g
```

- `[CLS]`는 `out.last_hidden_state[:, 0, :]`로 추출(`encode_cls`).
- 입력은 `DualTextDataset`(`__init__`에서 1회 토큰화)와 `make_dual_collator`(제주어/표준어
  각각 배치 최댓값으로 **동적 패딩**)로 공급.
- 학습 산출물: `torch.save(model.state_dict(), step3_dual_state.pt)`.

#### 2.A.4 KoELECTRA 브랜치 (Branch 1)

- 인코더: `monologg/koelectra-base-v3-discriminator`.
- 입력: **제주어 문장** 단일 텍스트 → `AutoModelForSequenceClassification(num_labels=7)`.
- `SingleTextDataset` + `DataCollatorWithPadding`(동적 패딩) 사용.
- 산출물: `run_step_single`이 `branch1_koelectra_model/`에 `model.save_pretrained` +
  `tokenizer.save_pretrained` 저장, `branch1_koelectra_probs.npz`(probs+labels) 저장.

#### 2.A.5 앙상블 (`run_step_ensemble`, Step 7)

```
p_ens = α · p_dual + (1 - α) · p_single
```

- 두 모델의 test `probs`(softmax 확률, `*_probs.npz`)를 불러와 α 그리드 서치.
- 기본 그리드: `np.linspace(0, 1, 21)` (0.00, 0.05, …, 1.00).
- 선택 기준: Test **F1-Macro** 최댓값. 선택된 α는 `step7_ensemble_report.txt`의 `Extra` 섹션과
  `Hyperparams`(`{"alpha": …}`)에 기록(서빙 시 이 값을 읽음, §6).
- 라벨 정합성 검증: `assert np.array_equal(labels_d, labels_s)` — 불일치 시 동일 seed 재실행 요구.

#### 2.A.6 하이퍼파라미터 (`DEFAULT_HP`, 발표본 슬라이드 18)

| 항목 | 값 | 비고 |
|------|----|------|
| Learning rate | `3e-5` | AdamW |
| Batch size | `64` | CLI `--batch-size`로 조정 가능 |
| Epochs | `7` | CLI `--epochs` |
| Max length | `64` (코드 기본) / 80 (발표본 표기) | 동적 패딩이므로 truncation 상한만 의미, 실제 토큰 최대 ~49 |
| Dropout | `0.2` | `hidden_dropout_prob` / Dual `nn.Dropout` |
| Weight decay | `0.01` | AdamW |
| Warmup ratio | `0.20` | `get_cosine_schedule_with_warmup` |
| Scheduler | Cosine | warmup 후 cosine decay |
| Label smoothing | `0.05` | cross_entropy |
| Grad clip | `1.0` | `clip_grad_norm_` |
| Seed | `42` | 분할·셔플 재현 |
| Mixed precision | bf16 (`--fp16`) | `torch.autocast(dtype=torch.bfloat16)`, MPS/CUDA 가속 |
| Noise filtering | 하위 30% 신뢰도 제거(발표본) | 본 데이터는 이미 정제됨 |

- **Hard Mining**(`_hard_mining_weights`): 1 epoch는 정상 학습, 이후 매 epoch loss 상위 20%
  (`top_pct=0.20`) 샘플 가중치 `alpha=2.0`로 `WeightedRandomSampler` 재구성. loss 계산은
  반드시 `shuffle=False` 로더로 수행(가중치-샘플 인덱스 정합).

**목표 F1-Macro = 0.84** (발표본 0.8404 재현).

### 2.B 번역 (신규, `src/translation/`)

#### 2.B.1 접근

사전학습 한국어 seq2seq **KoBART**(`gogamza/kobart-base-v2`, 약 124M)를 병렬 코퍼스로
파인튜닝한다. 한국어 BART라 제주어↔표준어처럼 표면형이 유사한 변환에 적합하고, M4에서
학습이 현실적이다.

#### 2.B.2 양방향 단일 모델 + 방향 prefix

한 모델로 두 방향을 학습한다. 소스 앞에 방향 토큰을 붙인다.

| 방향 | 입력(source) | 타깃(target) |
|------|--------------|--------------|
| 제주어→표준어 (`j2s`) | `"표준어로: <제주어 문장>"` | `<표준어 문장>` |
| 표준어→제주어 (`s2j`) | `"제주어로: <표준어 문장>"` | `<제주어 문장>` |

각 병렬쌍이 양방향 2개 샘플을 만들어 **데이터가 2배**가 되고 표현을 공유한다.

#### 2.B.3 데이터 구성

| 항목 | 값 |
|------|------|
| 소스 | `merged_jeju_balanced_sorted.xlsx`의 `제주어 문장` / `표준어 문장` (127k 쌍) |
| 증강 | 각 쌍 → `j2s` + `s2j` 2 샘플 (≈254k 학습 샘플) |
| 분할 | seed=42 고정, **8:1:1** (Train/Val/Test). 번역은 감정과 분할 정합성 불필요 |
| 결측 처리 | 두 문장 컬럼 NA 제거 |

#### 2.B.4 평가

- `sacrebleu`로 **BLEU + chrF** 측정.
- 베이스라인: 무번역 copy(입력 그대로 출력) 또는 짧은 학습 모델. 베이스라인 대비 향상 보고.
- 짧은 발화 위주라 BLEU 변동이 크므로 **chrF 병행** 보고(리스크 대응).

#### 2.B.5 구현 명세

| 파일 | 역할 | 핵심 인터페이스 |
|------|------|-----------------|
| `src/translation/train_translation.py` | KoBART 파인튜닝 | CLI: `--direction both`(기본), `--epochs`, `--batch-size`, `--data`, `--output-dir` |
| `src/translation/translate.py` | 추론 래퍼 | `translate(text: str, direction: "j2s"|"s2j") -> str` (prefix 부착 + beam search) |
| `scripts/run_translation.sh` | 학습 러너 | **감정 goal run 종료 후** 실행(단일 GPU 경합 방지) |

- 학습 산출물: `results_translation/kobart_j2s2j/`에 `model.save_pretrained` +
  `tokenizer.save_pretrained`.
- 추론: beam search(예: `num_beams=4`), 최대 생성 길이는 입력 토큰 길이에 여유를 둠.

### 2.C 챗봇 앱 (신규, `app/`)

#### 2.C.1 백엔드 (FastAPI, `app/main.py`)

- `uvicorn app.main:app`로 기동. **시작 시(lifespan) 모델 1회 로드**, 디바이스 MPS/CPU 자동 선택.
- 모델 아티팩트 경로는 **환경변수/설정으로 주입**. 모델이 없으면 **graceful degradation**:
  로드 가능한 기능만 노출하고, 불가한 엔드포인트는 명확한 에러를 반환.

엔드포인트(요약, 상세 스키마는 §4):

| 메서드 | 경로 | 요청 | 응답 |
|--------|------|------|------|
| GET | `/` | - | 정적 프론트(index.html) |
| POST | `/api/translate` | `{ text, direction:"j2s"|"s2j" }` | `{ translation }` |
| POST | `/api/emotion` | `{ text }` | `{ label, label_id, scores[7] }` |
| POST | `/api/chat` | `{ text, direction? }` | `{ translation, emotion:{ label, label_id, scores } }` |

`/api/chat` 동작:
- `direction` 미지정 또는 `"j2s"`: 입력을 **제주어**로 보고 표준어 번역 + 감정 분석을 함께 반환.
- `direction="s2j"`: 표준어→제주어 **번역만** 반환(감정 생략 또는 null).

#### 2.C.2 서비스 계층 (의존성 주입, 테스트 가능하게 분리)

**`app/services/emotion_service.py`**
- Dual-Gated KR-BERT를 `DualGatedClassifier` 인스턴스로 만들고 `step3_dual_state.pt`
  `state_dict` 로드.
- KoELECTRA를 `branch1_koelectra_model/`에서 `from_pretrained`로 로드.
- 추론: `p_dual`, `p_single`을 softmax 확률로 구해 `p_ens = α·p_dual + (1-α)·p_single`
  (α는 ensemble report에서 읽음). `argmax` → `label_id`/`label`, `p_ens` → `scores[7]`.
- 모델 없으면 명확한 에러(graceful degradation).

**`app/services/translation_service.py`**
- `results_translation/kobart_j2s2j/`에서 KoBART + 토크나이저 로드.
- `translate(text, direction)`: 방향 prefix 부착 → beam search 생성.

> 서비스 계층을 라우팅에서 분리해, 모델 없이 mock 주입으로 단위 테스트가 가능하도록 한다(§8).

#### 2.C.3 프론트 (정적 SPA)

- 빌드 단계 없는 단일 페이지: `app/static/index.html` + `app.js` + `style.css`.
- 구성요소: 채팅 말풍선, 방향 토글(`j2s`/`s2j`), **감정 뱃지**(감정별 이모지+색), 신뢰도 막대.
- 감정 뱃지 매핑 예(7클래스):

| label_id | label | 예시 이모지 |
|---|---|---|
| 0 | 중립 | 😐 |
| 1 | 슬픔 | 😢 |
| 2 | 행복 | 😊 |
| 3 | 분노 | 😠 |
| 4 | 놀람 | 😲 |
| 5 | 공포 | 😱 |
| 6 | 혐오 | 🤢 |

- `fetch`로 `/api/chat`(또는 `/api/translate`·`/api/emotion`) 호출, 응답을 말풍선에 렌더.

## 6. 모델 아티팩트 및 서빙

### 6.1 아티팩트 형식

| 모델 | 저장 방식 | 경로 | 로드 방법 |
|------|-----------|------|-----------|
| Dual-Gated KR-BERT | `torch.save(model.state_dict(), …)` | `results_kr_bert/step3_dual_state.pt` | `DualGatedClassifier(encoder)` 인스턴스 생성 후 `load_state_dict` |
| KoELECTRA | `save_pretrained` + 토크나이저 | `results_kr_bert/branch1_koelectra_model/` | `AutoModelForSequenceClassification.from_pretrained` |
| 앙상블 α | report 텍스트 | `results_kr_bert/step7_ensemble_report.txt` | `Hyperparams`/`Extra`의 `alpha` 파싱 |
| KoBART(번역) | `save_pretrained` + 토크나이저 | `results_translation/kobart_j2s2j/` | `AutoModelForSeq2SeqLM.from_pretrained` |

- **저장 코드 정합**: 현재 `run_step_single`은 보고서·probs와 함께
  `model.save_pretrained(step_name_model)` + `tokenizer.save_pretrained`를 수행하므로
  KoELECTRA 가중치가 서빙용으로 저장된다(`src/jeju_kr_bert.py` 참조). Dual은
  `torch.save(state_dict, step3_dual_state.pt)`로 별도 저장.
- KoELECTRA 단계는 dual 학습 뒤 별도 프로세스로 실행되므로, 현재 소스 기준으로 그때 자동 저장된다.

### 6.2 앙상블 α 출처

- Step 7(`run_step_ensemble`)에서 그리드 서치로 F1-Macro 최대 α를 선택하고
  `step7_ensemble_report.txt`에 기록.
- 서빙 시 `emotion_service`가 이 α를 읽어 `p_ens = α·p_dual + (1-α)·p_single`로 결합.
- α 파일이 없으면 안전 기본값(예: 0.5) 사용 후 경고 로그.

### 6.3 서빙 시 메모리

KR-BERT + KoELECTRA + KoBART 동시 로드. 16GB 내에서 관리하되, 필요 시 **지연 로드**(요청
시점 로드)로 메모리 압박을 완화한다.

| 데이터 | `pandas`, `numpy`, `openpyxl`(xlsx) |

### 7.2 실행 방법

**감정 학습 (GPU, 직렬)**

```bash
DATA="merged_jeju_balanced_sorted.xlsx"
# Dual-Gated KR-BERT (Step 3)
python3 src/jeju_kr_bert.py --step dual      --data "$DATA" --epochs 7 --batch-size 64 --fp16
# KoELECTRA (Branch 1)
python3 src/jeju_kr_bert.py --step koelectra --data "$DATA" --epochs 7 --batch-size 64 --fp16
# 앙상블 (Step 7) — 두 모델 test probs로 α 그리드 서치
python3 src/jeju_kr_bert.py --step ensemble \
  --dual-probs   results_kr_bert/step3_dual_probs.npz \
  --single-probs results_kr_bert/branch1_koelectra_probs.npz
```

**번역 학습 (GPU, 감정 완료 후)**

```bash
bash scripts/run_translation.sh          # 또는
python3 src/translation/train_translation.py --direction both --data "$DATA" --epochs 3
```

**앱 실행 (추론)**

```bash
# 모델 경로 환경변수 주입 후
uvicorn app.main:app --reload --port 8000
# 브라우저: http://localhost:8000/
```

> 스모크: 감정 학습은 `--limit N`으로 작은 부분집합만 사용해 빠르게 파이프라인을 점검.

## 부록 A. 핵심 식별자 요약 (코드 정합)

```
