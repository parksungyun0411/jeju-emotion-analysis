📝 2. 텍스트 데이터 특성
============================================================

🔸 표준어 vs 제주어 비교
------------------------------------------------------------
- 표준어 문장: 일반적인 한국어 문장
- 제주어 문장: 제주도 방언으로 표현된 문장
- 동일한 내용을 두 언어로 표현한 쌍 데이터

예시:
표준어: "정말 기뻐요"
제주어: "정말 기뻐유"

표준어: "슬프네요"
제주어: "슬프네유"

🔸 감정 분류 기준 (7개 클래스)
------------------------------------------------------------
0: 중립 - 감정이 없는 객관적 서술
1: 슬픔 - 우울, 아쉬움, 실망, 그리움
2: 행복 - 즐거움, 기쁨, 만족, 희망
3: 분노 - 화남, 짜증, 불만, 적대감
4: 놀람 - 깜짝, 당황, 신기함, 충격
5: 공포 - 무서움, 두려움, 불안, 걱정
6: 혐오 - 싫음, 역겨움, 거부감, 불쾌함

🔧 3. 벡터화 방법: TF-IDF
============================================================

🔸 TF-IDF (Term Frequency-Inverse Document Frequency)
------------------------------------------------------------
- 목적: 텍스트를 수치 벡터로 변환
- 특징: 단어의 중요도를 반영한 벡터화

🔸 설정 파라미터
------------------------------------------------------------
- max_features: 30,000 (최대 특성 수)
- ngram_range: (1, 3) (1-gram, 2-gram, 3-gram)
- stop_words: None (한국어 불용어 처리 안함)
- lowercase: False (대소문자 구분 안함)
- sublinear_tf: True (로그 스케일링 적용)
- use_idf: True (IDF 가중치 사용)
- smooth_idf: True (IDF 스무딩 적용)

🔸 N-gram 설명
------------------------------------------------------------
1-gram (단어): "정말", "기뻐", "유"
2-gram (단어쌍): "정말 기뻐", "기뻐 유"
3-gram (단어 3개): "정말 기뻐 유"

장점: 문맥 정보 보존
단점: 특성 수 급증, 희소성 문제

🤖 4. 머신러닝 모델
============================================================

🔸 Logistic Regression (로지스틱 회귀)
------------------------------------------------------------
- 원리: 선형 분류기, 시그모이드 함수 사용
- 장점: 해석 가능, 빠른 학습, 안정적
- 단점: 선형 관계만 학습 가능
- 파라미터: C=1.0, max_iter=1000

🔸 SVM (Support Vector Machine)
------------------------------------------------------------
- 원리: 최적의 분류 경계면 찾기
- 장점: 고차원 데이터에 강함, 과적합 방지
- 단점: 대용량 데이터에 느림, 해석 어려움
- 파라미터: kernel='linear', probability=True

🔸 Random Forest (랜덤 포레스트)
------------------------------------------------------------
- 원리: 여러 의사결정나무의 앙상블
- 장점: 과적합 방지, 특성 중요도 제공
- 단점: 메모리 사용량 큼, 해석 어려움
- 파라미터: n_estimators=100, max_depth=None

🔸 Naive Bayes (나이브 베이즈)
------------------------------------------------------------
- 원리: 베이즈 정리 기반 확률적 분류
- 장점: 빠른 학습, 작은 데이터셋에 효과적
- 단점: 특성 독립성 가정 (비현실적)
- 파라미터: alpha=1.0 (라플라스 스무딩)

📊 5. 평가 방법론
============================================================

🔸 데이터 분할
------------------------------------------------------------
- 훈련 데이터: 80% (모델 학습용)
- 테스트 데이터: 20% (성능 평가용)
- 분할 방법: stratify=y (클래스 비율 유지)
- 랜덤 시드: random_state=42 (재현 가능)

🔸 평가 지표
------------------------------------------------------------

1) Accuracy (정확도)
- 정의: 전체 예측 중 맞춘 비율
- 공식: (TP + TN) / (TP + TN + FP + FN)
- 특징: 직관적이지만 불균형 데이터에 취약

2) F1-Macro (매크로 평균 F1)
- 정의: 각 클래스별 F1의 평균
- 공식: (F1_class1 + F1_class2 + ... + F1_class7) / 7
- 특징: 모든 클래스를 동등하게 평가

3) F1-Weighted (가중 평균 F1)
- 정의: 클래스별 샘플 수에 비례한 가중 평균
- 공식: Σ(F1_i × support_i) / Σ(support_i)
- 특징: 다수 클래스에 더 가중치 부여

🔸 F1-Score 상세 설명
------------------------------------------------------------
- Precision (정밀도): 예측한 양성 중 실제 양성 비율
- Recall (재현율): 실제 양성 중 예측한 양성 비율
- F1-Score: Precision과 Recall의 조화평균

공식: F1 = 2 × (Precision × Recall) / (Precision + Recall)

🔸 Support (지지도)
------------------------------------------------------------
- 정의: 각 클래스의 실제 샘플 수
- 용도: 클래스별 데이터 분포 확인
- 중요성: 불균형 데이터 분석에 필수

📊 7. 실험 결과 해석 가이드
============================================================

🔸 성능 지표 해석
------------------------------------------------------------
- Accuracy > 50%: 랜덤 추측보다 나음
- F1-Macro > 0.3: 7개 클래스 균형 분류에서 양호
- F1-Weighted > 0.4: 불균형 데이터에서 양호

🔸 모델별 특성
------------------------------------------------------------
- SVM: 불균형 데이터에서 우수
- Naive Bayes: 균형 데이터에서 우수
- Random Forest: 안정적이지만 성능 낮음
- Logistic Regression: 중간 수준의 안정적 성능

🔸 데이터셋별 특성
------------------------------------------------------------
- 불균형 데이터: 높은 정확도, 낮은 F1-Macro
- 균형 데이터: 낮은 정확도, 높은 F1-Macro
- 차이점: 클래스 분포의 영향
