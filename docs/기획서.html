1프로젝트 개요

앱 이름 (확정)탐라 (Tamna) — 제주의 옛 이름에서 따온 가제로, 제주의 정체성과 "모든 것을 품는다"는 의미를 함께 담는다.
한 줄 소개제주 여행·생활에 필요한 모든 것 — 제주어 번역/감정분석, 맛집, 명소·체험, AI 여행경로, 커뮤니티 — 을 한 앱에 담은 제주 올인원 모바일 웹앱
형태모바일 우선 반응형 웹앱 (하단 4탭 네비게이션), 추후 PWA/네이티브 래핑 확장
분류위치기반 서비스(LBS) + NLP(소멸위기언어) + 커뮤니티 + 생성형 AI
대상 학기1학기 번역기 제안 → 2학기 감정분류 → 통합 올인원 앱(현재)

추진 배경

제주어는 유네스코가 지정한 소멸위기언어(2010년, "심각한 위기" 등급)이며, 표준어 중심 환경과 세대 간 단절로 젊은 세대는 제주어를 이해할 기회가 급격히 줄고 있다. 본 프로젝트는 이 문제의식에서 출발해 1·2학기 동안 제주어 번역기 제안과 제주어 감정분류(F1-Macro 0.84)라는 ML 자산을 축적했다.

한편 실제 사용자(여행객·이주자)를 만나며 확인한 것은, 제주에서 필요한 정보(언어·맛집·명소·체험·여행 일정·현지 꿀팁)가 여러 앱과 블로그·지도·검색에 파편적으로 흩어져 있다는 점이다. 이에 본 프로젝트는 기존 제주어 ML 성과를 하나의 탭으로 흡수하고, 그 위에 맛집 지도·명소 큐레이션·AI 여행경로·커뮤니티를 더해, 제주에 관한 모든 것을 한 곳에서 해결하는 올인원 앱 "탐라"로 범위를 확대한다.

2문제 정의 및 필요성

문제

#문제설명
1정보 분산제주 여행/생활 정보(맛집·명소·체험·교통·꿀팁)가 지도앱, 블로그, 검색, 커뮤니티 등에 흩어져 있어 사용자가 앱을 계속 바꿔 가며 정보를 모아야 한다.
2언어 장벽제주어는 어휘·어미가 표준어와 크게 달라 비제주민·젊은 세대가 현지인의 말, 간판, 메시지를 이해하기 어렵다.
3일정 설계의 어려움가고 싶은 명소를 골라도 동선·일자별 배치를 짜는 것은 수작업이며 시간이 많이 든다.
4현지 정보 부족검증된 현지 꿀팁·질의응답을 모을 공간이 없어, 같은 질문이 분산된 채 반복된다.
5성과의 파편화번역·감정분류 ML 성과가 연구로만 남아, 사용자가 실제로 손에 쥐고 쓰는 제품으로 이어지지 못했다.

필요성

관점필요성
사용자 편익흩어진 제주 정보를 하나의 앱에서 끊김 없이 — 검색·이해·계획·소통까지 — 해결
문화 보존소멸위기언어인 제주어의 디지털 접근성을 높여 학습·소통 촉진
기술 가치저자원 방언 NLP + 위치기반 서비스 + 생성형 AI를 결합한 실증 사례
활용성관광·콘텐츠·교육·언어연구 등 다방면 응용 기반 마련
기존 sklearn TF-IDF 베이스라인은 제주어 감정분류에서 F1-Macro 0.30에 그쳤으나, 딥러닝 전환(KR-BERT 단일 0.76 → 앙상블 0.84)으로 실용 수준을 확보했다. 즉, 본 앱의 탭 1은 이미 검증된 기술 토대 위에서 동작한다.

3목표 및 범위

목표

제주 여행·생활에 필요한 핵심 기능을 4개 탭으로 통합한 모바일 웹앱 "탐라"를 완성한다. 각 탭은 독립적으로 동작하면서도 사용자 여정 속에서 자연스럽게 이어진다.

4탭 기능 상세

탭이름핵심 기능

사용 시나리오 (탭을 넘나드는 여정)

시나리오 1 — 여행객 지민의 첫 제주 여행 (탭3 → 탭2 → 탭1)

지민은 3박4일 일정을 짜야 한다.
  1. 탭3 (명소·체험)에서 가고 싶은 명소(성산일출봉, 카멜리아힐 등)와 체험(승마)을 지도에서 골라 다중 선택한다.
  2. AI 여행경로 생성을 누르고 "3박4일, 렌터카, 자연·포토 위주"를 입력하니, 일자별 동선이 지도에 폴리라인으로 그려진다.
  3. 점심 시간대가 비자, 탭2 (맛집)으로 이동해 동선 근처 맛집을 검색하고 길찾기로 연결한다.
  4. 현지 식당 간판에서 본 "혼저 옵서예" 문구가 궁금해 탭1 (제주어)에 입력하니 "어서 오세요"로 번역된다.

시나리오 2 — 이주자 현우의 현지 정착 (탭4 → 탭1 → 탭4)

현우는 카페 운영 중 궁금한 점을 해결하고 싶다.
  1. 탭4 (커뮤니티)에 Google 계정으로 로그인하고, "겨울철 제주 카페 손님이 줄 때 팁?"을 질문 글로 올린다.
  2. 단골 어르신이 남긴 제주어 메시지의 정서가 궁금해 탭1에 입력하니 번역과 함께 "🙂 행복(0.71) — 애정 어린 당부"로 분석된다.
  3. 다시 탭4 마이페이지에서 내가 쓴 글·받은 댓글을 확인하고, 유용한 꿀팁 글을 북마크한다.

시나리오 3 — 학습자 서연의 콘텐츠 제작 (탭1 → 탭4)

서연은 제주어 자막 영상을 만든다.
  1. 탭1 (제주어) 방향 토글을 "표준어→제주어"로 두고 "어서 오세요, 천천히 구경하세요"를 입력해 "혼저 옵서, 놀멍 보멍 갑서" 자막 후보를 얻는다.
  2. 역방향(제주어→표준어)으로 다시 검수하고, 감정분석으로 정서 톤을 확인한다.
  3. 탭4 (커뮤니티) 후기 게시판에 제작 과정과 결과물을 공유해 피드백을 받는다.

5탭별 주요 기능 상세

탭 1 — 제주어 (번역 + 감정)

  • 양방향 번역: 입력창 + 방향 토글(제주어→표준어 / 표준어→제주어). 단일 KoBART 모델에 방향 prefix를 부여해 두 방향을 함께 처리.
  • 감정 분석(7종): 제주어 문장의 감정을 중립 · 슬픔 · 행복 · 분노 · 놀람 · 공포 · 혐오 7종으로 분류, 신뢰도(확률)를 함께 제시. 이모지·색상 뱃지와 신뢰도 막대로 직관적 표시(단정 대신 분포 제시).
  • 결과 카드: 번역문 + 감정 뱃지·신뢰도를 한 카드에 표시.
  • graceful degradation: ML 미준비 시 "모델 학습 중" 안내(503).
  • 데이터 경로: Next 서버 라우트 /api/translate, /api/emotion이 FastAPI ML 서비스(ML_SERVICE_URL)를 프록시.

탭 2 — 맛집 (지도 · 길찾기)

  • 지도 표시: 카카오 지도에 제주 맛집을 마커로 표시. 데이터 출처는 (a) Kakao Local 키워드/카테고리(FD6 음식점) 검색, (b) 큐레이션 seed JSON(restaurants.seed.json) 폴백.
  • 상세 정보: 마커 클릭 시 바텀시트에 이름/카테고리/주소/평점/사진(placeholder).
  • 길찾기: 카카오 지도 길찾기 링크로 연결.
  • 탐색: 지역·카테고리 필터, 키워드 검색.

탭 3 — 명소·체험 (지도 + AI 여행경로)

  • 큐레이션: 명소/체험을 카테고리(자연/문화/체험/포토)로 큐레이션(seed JSON attractions.seed.json).
  • 지도·상세·길찾기: 마커 표시, 상세 바텀시트, 길찾기 링크.
  • 다중 선택(장바구니): 가고 싶은 장소를 여러 개 담기.
  • AI 여행경로 생성: 선택 장소 + 사용자 조건(일수 / 이동수단 / 취향)을 서버 라우트 /api/route가 Claude API로 보내 일자별 동선 일정을 생성. 결과를 지도에 폴리라인으로 표시.

탭 4 — 커뮤니티 + 로그인 + 마이페이지

  • 게시판: 카테고리(질문/꿀팁/후기). 목록·상세·작성·댓글·좋아요. 작성/댓글은 로그인 필요.
  • 인증: NextAuth.js (Google + Kakao provider). 세션은 DB(Prisma adapter)에 저장.
  • 마이페이지: 프로필(이름/이메일/아바타), 내 글·댓글, 좋아요·북마크, 로그아웃.

6기술 개요

[모바일 웹 브라우저] ── Next.js 15 (webapp/) ── 하단 4탭 네비
   ├─ 탭1 제주어   → /api/translate · /api/emotion ─proxy→ [FastAPI ML]
   ├─ 탭2 맛집     → Kakao Maps SDK + /api/places  ─→ [Kakao Local]
   ├─ 탭3 명소·체험 → Kakao Maps + /api/route(AI)    ─→ [Claude API]
   ├─ 탭4 커뮤니티  → /api/posts ...                 ─→ [Prisma/DB]
   └─ /api/auth/*  → NextAuth (Google, Kakao)
[FastAPI ML 서비스] 번역(KoBART) + 감정(Dual KR-BERT + KoELECTRA 앙상블)

설계 원칙

  • 서비스 분리: Next.js(웹앱·커뮤니티·인증·AI라우트)와 Python FastAPI(ML 추론)는 별도 프로세스. Next.js가 ML_SERVICE_URL로 ML을 호출.
  • 탭 독립성: 각 탭은 다른 탭 없이도 동작하도록 점진 구현·점진 출시.
  • graceful degradation: 외부 키(Kakao/OAuth/Claude)·모델이 없으면 seed/mock·안내로 동작. 앱 자체는 키 없이도 기동·탐색 가능. .env.example로 키 명세.
